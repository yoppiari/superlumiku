### Shannon Entropy Calculation

**Formula**: H = -Œ£(p(x) √ó log‚ÇÇ(p(x)))

**Where**:
- H = entropy in bits per character
- p(x) = probability of character x
- Higher value = more random/secure

**Examples**:
- `aaaaaaaa`: 0.0 bits/char ‚Üí CRITICAL RISK
- `password`: 2.75 bits/char ‚Üí CRITICAL RISK
- `change-this-secret-key`: 3.16 bits/char ‚Üí HIGH RISK (also blacklisted)
- `MyApp2024!Secret`: 3.62 bits/char ‚Üí MEDIUM RISK (borderline)
- `crypto.randomBytes(32).hex`: 4.0 bits/char ‚Üí SECURE

**Minimum Required**: 3.5 bits/char

### Secret Generation

```typescript
import { randomBytes } from 'crypto'

function generateSecureSecret(length: number = 32): string {
  return randomBytes(length).toString('hex')
}

// Usage:
const secret = generateSecureSecret(32)
// Output: 64-character hex string
// Example: "8d024b643b19488486b7aa1ab4dd0a1cf1566d76603252f659ce9bd905fc06e9"
```

**Security**:
- Uses Node.js crypto module (cryptographically secure)
- 32 bytes = 256 bits of entropy
- Hex encoding = 64 characters
- Search space: 2^256 combinations
- Brute force time at 1 trillion attempts/second: ~10^59 years

## Security Best Practices Implemented

### DO (Implemented) ‚úÖ

- ‚úÖ Use cryptographically secure random generation (`crypto.randomBytes()`)
- ‚úÖ Minimum 32 bytes (64 hex chars) for production
- ‚úÖ Validate secret strength on startup
- ‚úÖ Fail fast if requirements not met
- ‚úÖ Store in environment variables only
- ‚úÖ Never commit to version control
- ‚úÖ Different secrets per environment
- ‚úÖ Log metadata only, never the secret
- ‚úÖ Clear error messages with remediation steps
- ‚úÖ Comprehensive documentation
- ‚úÖ Architecture ready for secret rotation

### DON'T (Prevented) ‚ùå

- ‚ùå Hard-coded secrets ‚Üí Prevented by validation
- ‚ùå Default secrets ‚Üí Blacklisted and rejected
- ‚ùå Weak secrets ‚Üí Entropy check enforces strength
- ‚ùå Short secrets ‚Üí Length validation enforces minimum
- ‚ùå Same secret across environments ‚Üí Documentation emphasizes separation
- ‚ùå Secret exposure ‚Üí Validation logs only metadata
- ‚ùå Silent failures ‚Üí Fail fast with clear errors
